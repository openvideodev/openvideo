title: Effects

Effects add visual flair to your video compositions. In OpenVideo Core, effects are stored as part of a clip's data and applied via commands. They use GLSL shaders for real-time, GPU-accelerated visual transformations.

## Overview

Effects in OpenVideo are:

- **GLSL-based**: Run on the GPU for smooth performance
- **Time-aware**: Can be applied to specific time ranges within a clip
- **Parameterized**: Many effects accept custom values for fine-tuning
- **Stackable**: Multiple effects can be applied to a single clip
- **Reversible**: All effect changes are undoable via the command system

## Effect Structure

Each effect follows this structure:

```ts
interface Effect {
  id: string; // Unique identifier
  key: string; // Effect type identifier
  startTime: number; // When effect starts (microseconds from clip start)
  duration: number; // How long effect lasts (microseconds)
  values?: object; // Effect-specific parameters (optional)
}
```

## Applying Effects

### Basic Effect Application

```ts
import { core, nanoid } from "@openvideo/core";

// Apply a sepia filter to a clip
core.clip.update(clipId, {
  effects: [
    {
      id: nanoid(),
      key: "sepia",
      startTime: 0,
      duration: 5_000_000, // 5 seconds
    },
  ],
});
```

### Time-Scoped Effects

Apply effects to specific portions of a clip:

```ts
// Grayscale only at the beginning, then fade to color
core.clip.update(clipId, {
  effects: [
    {
      id: "intro-gray",
      key: "grayscale",
      startTime: 0,
      duration: 2_000_000, // First 2 seconds
    },
  ],
});
```

### Stacking Multiple Effects

```ts
core.clip.update(clipId, {
  effects: [
    {
      id: "vintage",
      key: "sepia",
      startTime: 0,
      duration: 10_000_000,
    },
    {
      id: "vignette",
      key: "vignette",
      startTime: 0,
      duration: 10_000_000,
    },
    {
      id: "film-look",
      key: "filmStripPro",
      startTime: 1_000_000,
      duration: 8_000_000,
    },
  ],
});
```

## Effect Values and Parameters

Many effects accept custom values to control their appearance:

```ts
core.clip.update(clipId, {
  effects: [
    {
      id: "custom-glow",
      key: "glowFilter",
      startTime: 0,
      duration: 5_000_000,
      values: {
        distance: 20, // Glow distance
        outerStrength: 3, // Outer glow intensity
        innerStrength: 1, // Inner glow intensity
        color: "#ff0000", // Glow color
      },
    },
    {
      id: "pixelated",
      key: "pixelate",
      startTime: 2_000_000,
      duration: 3_000_000,
      values: {
        pixelSize: 20, // Pixel block size
      },
    },
  ],
});
```

## Managing Effects

### Adding Effects

```ts
// Add to existing effects
const currentEffects = core.store.getState().clips[clipId].effects || [];
core.clip.update(clipId, {
  effects: [
    ...currentEffects,
    {
      id: nanoid(),
      key: "vignette",
      startTime: 0,
      duration: 10_000_000,
    },
  ],
});
```

### Updating Effect Values

```ts
// Update specific effect values
const currentEffects = core.store.getState().clips[clipId].effects;
core.clip.update(clipId, {
  effects: currentEffects.map((effect) =>
    effect.id === "custom-glow"
      ? { ...effect, values: { ...effect.values, color: "#00ff00" } }
      : effect,
  ),
});
```

### Removing Effects

```ts
// Remove all effects
core.clip.update(clipId, { effects: [] });

// Remove specific effect by ID
const currentEffects = core.store.getState().clips[clipId].effects || [];
core.clip.update(clipId, {
  effects: currentEffects.filter((e) => e.id !== "vintage"),
});
```

## Batch Effect Operations

Apply effects to multiple clips at once:

```ts
// Apply grayscale to all selected clips
const state = core.store.getState();
const selectedIds = state.selectedIds;

core.execute({
  id: nanoid(),
  type: "clip.update",
  payload: selectedIds.map((id) => ({
    id,
    updates: {
      effects: [
        ...(state.clips[id].effects || []),
        {
          id: nanoid(),
          key: "grayscale",
          startTime: 0,
          duration: state.clips[id].timing?.duration || 5_000_000,
        },
      ],
    },
  })),
});
```

## Custom Effects

Register your own GLSL shaders for use in the effects system:

```ts
import { registerCustomEffect } from "@openvideo/core";

registerCustomEffect("channelSwap", {
  label: "Channel Swap",
  fragment: `
    vec4 mainImage(vec2 p) {
      vec4 color = getTextureColor(p);
      // Swap red and green channels
      return vec4(color.g, color.r, color.b, color.a);
    }
  `,
  uniforms: {
    intensity: { value: 1.0, type: "f32" },
  },
});

// Use the custom effect
core.clip.update(clipId, {
  effects: [
    {
      id: nanoid(),
      key: "channelSwap",
      startTime: 0,
      duration: 5_000_000,
      values: { intensity: 0.8 },
    },
  ],
});
```

### GlEffect Interface

```ts
interface GlEffect {
  label: string; // Display name
  fragment: string; // GLSL fragment shader
  uniforms?: Record<
    string,
    {
      // Default uniform values
      value: number;
      type: "f32" | "i32";
    }
  >;
  previewStatic?: string; // Static preview image URL
  previewDynamic?: string; // Animated preview URL
}
```

## Best Practices

1. **Use effect IDs**: Always provide unique IDs for managing individual effects
2. **Time in microseconds**: `startTime` and `duration` are in μs
3. **Stack purposefully**: Order matters — effects are applied sequentially
4. **Reuse values**: Define common value sets as constants for consistency
5. **Consider performance**: Many heavy effects on multiple clips can impact playback
6. **Preview before export**: Effects render in real-time — preview to ensure they look right
7. **Combine with animations**: Use effects alongside animations for rich visuals
